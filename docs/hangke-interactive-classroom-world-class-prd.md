# 航科互动课堂世界级产品 PRD

更新时间：2026-03-20

## 1. 定位

航科互动课堂不再被定义为“老师上课时临时生成的一节 AI 课堂”，而是升级为一套覆盖学生自主学习、教师课堂共学、数字人教学分身、学校治理运营的教学操作系统。

产品主张：

- 对学生：把预习、巩固、兴趣探索、课堂回看做成可反复使用的自主学习工作台
- 对教师：把一次备课沉淀成可发布、可复用、可导出的学习资产
- 对学校：把互动课堂纳入统一治理、效果评估和教务策略配置
- 对平台：把模型能力封装成可审计、可配置、可扩张的教育 AI 能力栈

## 2. 北极星目标

我们不把“生成成功率”当作北极星，而把“真实学习效果”和“持续复用率”作为产品核心指标。

北极星指标：

- 学生 7 日自主回访率
- 学生单次学习闭环完成率
- 教师学习包复用率
- 互动课堂带来的薄弱点收口率
- 预习后课堂参与提升率
- 兴趣课堂转化为正式学习任务的比例

## 3. 用户与核心价值

### 3.1 学生

学生使用互动课堂，不是为了获得一个答案，而是为了获得一个带着自己学下去的学习过程。

学生价值：

- 有明确入口，不需要自己组织学习脚本
- AI 会先诊断、再讲解、再练习、再复盘
- 支持课前预习、课后巩固、兴趣探索、课堂回看
- 可听、可看、可互动、可回放、可导出

### 3.2 教师

教师不再只是在课堂上调用一次工具，而是在建立自己的教学数字分身和可复用学习资产库。

教师价值：

- 一键生成数字人形象、音色和教学风格
- 课堂内容可同时支持整班观看与学生个体自学
- 一次备课可产出预习包、巩固包、兴趣包、回看包
- 内容导出后仍能回流治理和效果分析

### 3.3 学校管理员

学校端关注的不是“用了多少次”，而是“是否带来了真实教学覆盖与效果提升”。

学校价值：

- 统一管理 API key、模型策略、权限边界、导出策略
- 统一查看教师采用、学生自主使用、班级扩散、资源沉淀
- 统一衡量互动课堂对排课、作业、课堂执行的联动效果
- 可审计、可配置、可导出、可规模化复制

## 4. 产品结构

### 4.1 学生自主学习工作台

统一入口固定为四种模式：

1. 课前预习
2. 学科巩固
3. 兴趣探索
4. 课堂回看

四种模式共享同一套学习画像、目标、错题、资料和学习记忆。

### 4.2 教师学习包工厂

教师端围绕“主题/班级/数字人”生成可复用学习资产：

- 共学课堂包
- 课前预习包
- 课后巩固包
- 兴趣拓展包
- 课堂回看包

### 4.3 学校治理与策略后台

学校后台负责统一控制：

- API key 与 provider 配置
- 模型路由与年级策略
- 未成年人帮助强度
- 导出权限与资源沉淀策略
- 采用效果与学习成效分析

## 5. 设计原则

### 5.1 学习优先，不以答案替代思考

产品默认应优先采用启发式、苏格拉底式和分步式讲解，避免让学生直接跳到结果。

### 5.2 任务优先，不以空白聊天框为入口

学生首先选择“我要预习 / 我要巩固 / 我要探索 / 我要回看”，而不是直接面对一个聊天输入框。

### 5.3 资产优先，不做一次性内容

每次生成都要尽量沉淀为老师可复用、学校可治理、学生可二次回看的学习资产。

### 5.4 治理优先，不做不可控能力

任何高价值能力都必须能被学校端配置、审计、导出和效果评估。

### 5.5 效果优先，不只追求时长和互动

我们衡量的是闭环完成率、薄弱点收口率、课堂参与提升、兴趣学习转化，而不是停留时长。

## 6. 学习闭环

学生自主学习统一走以下闭环：

1. 目标确认
2. 水平诊断
3. 分步讲解
4. 即时互动
5. 小练检验
6. 错因解释
7. 总结复盘
8. 下一步建议

四种模式的差异只体现在课堂编排策略，不改变闭环骨架。

## 7. 四种模式定义

### 7.1 课前预习

适用场景：

- 新课开始前 10 到 15 分钟
- 学生对新单元没有建立框架
- 老师需要学生带着问题进入课堂

核心体验：

- 激活旧知
- 建立知识主线
- 生成问题清单
- 用 2 到 3 个检测题确认预习质量

关键产出：

- 预习导读
- 问题清单
- 重点概念图
- 预习自测

### 7.2 学科巩固

适用场景：

- 错题后定向收口
- 考前专题补弱
- 作业暴露薄弱点后的快速修复

核心体验：

- 围绕一个薄弱点收口
- 强化分步讲解
- 立刻插入跟练
- 最后做迁移题检验

关键产出：

- 定向讲解
- 例题拆解
- 即时练习
- 错因总结卡

### 7.3 兴趣探索

适用场景：

- 课后延展
- 学科兴趣培养
- 跨学科主题探究

核心体验：

- 情境化导入
- 角色式讲解
- 项目式追问
- 让学生愿意继续探索

关键产出：

- 主题故事线
- 互动追问
- 拓展任务
- 兴趣探索卡

### 7.4 课堂回看

适用场景：

- 课后复盘
- 考前把主干再吃一遍
- 学生觉得“上课好像听懂了，但自己讲不出来”

核心体验：

- 回收课堂主线
- 提炼核心概念
- 把老师讲过的内容重新压缩成可复述结构
- 最后让学生自己说出来

关键产出：

- 课堂重点回看
- 关键题型回放
- 复述卡
- 课后追练入口

## 8. 数字人策略

数字人不是装饰，而是教师教学分身。

最小闭环能力：

- 动漫画像
- 音色和声音配置
- 教学风格模板
- 讲解协议
- 课堂脚本引用老师资料

数字人约束：

- 默认优先使用老师/学校资料
- 不允许跳过关键讲解直接给答案
- 高风险输出必须提示复核
- 所有生成行为进入学校后台审计

## 9. 学校治理指标

学校端围绕以下指标持续治理：

- 教师采用率
- 学生自主学习率
- 整班观看发布率
- 导出后复用率
- 资源包沉淀率
- 学习模式分布
- 班级扩散率
- 学生闭环完成率
- 薄弱点收口率

## 10. 当前项目落地路线

### Phase 1

- 完成学生自主学习工作台四模式入口
- 统一学习模式类型与治理标签
- 保持 AI 互动课堂启动链路兼容

### Phase 2

- 教师端学习包工厂
- 数字人创建与配置流程强化
- 回看/预习/巩固统一发布到班级

### Phase 3

- 学校后台加入模式级效果分析
- 接通作业、排课、课程模块、学生日程
- 学习效果治理从“使用统计”升级为“成效统计”

## 11. 本轮实施决定

为保证路线稳定，当前项目做出以下明确决策：

- 互动课堂品牌统一为“航科互动课堂”
- 学生自主学习是主链路，不是附属功能
- 世界级方向先从“任务化入口 + 学习闭环 + 学校治理”落地
- 技术实现优先保证统一类型、统一上下文、统一学校治理兼容
