# Week9 任务卡（P0 功能落地）

## 任务卡 1：今日必做统一任务队列

### 目标
- 学生端将作业、考试、错题复练、学习计划、挑战整合为同一“今日任务队列”，按优先级驱动执行。

### 前端
- 学生首页 `app/student/page.tsx`
  - “今日任务”卡片改为展示统一任务队列。
  - 展示任务状态（逾期/今日到期/进行中/待完成）和快速跳转入口。
  - “刷新学习计划”后同步刷新今日任务数据。

### 后端
- 新增 `GET /api/student/today-tasks`
  - 聚合来源：`/assignments`、`/exams`、`wrong-review`、`study plan`、`challenges`。
  - 输出：
    - `summary`：总数、必做数、逾期数、今日到期数、来源分布
    - `groups`：`mustDo / continueLearning / growth`
    - `tasks`：统一任务列表（含 `source/status/priority/href`）

### 数据
- 不新增持久化表，使用现有数据源聚合计算。
- 任务优先级规则在接口层统一计算（可后续抽为配置）。

### 验收标准
- 学生打开首页可看到跨模块混排的任务队列。
- 队列中“逾期/今日到期/进行中”任务优先展示。
- `GET /api/student/today-tasks` 在 API 测试中通过结构断言。

### 风险与回滚
- 风险：聚合接口响应延迟上升。
- 回滚：学生首页退回原“学习计划 + 错题复练摘要”展示逻辑。

## 任务卡 2：考试结果自动进入错题闭环

### 目标
- 学生在线考试提交后，错题自动写入 24h/72h/7d 复练队列，形成“考试 -> 复练”闭环。

### 前端
- 学生考试结果页增加提示文案：
  - “错题已加入今日复练清单”。
- 错题本首页可直接看到考试来源复练任务。

### 后端
- 在考试提交链路中接入错题入队：
  - 依据考试答题结果，错误题调用错题队列入队逻辑。
- 可选扩展字段：
  - `source: exam`
  - `sourcePaperId`
  - `sourceSubmittedAt`

### 数据
- 复用 `wrong_review_items`。
- 增加来源字段时需同步 DB schema / JSON 结构兼容。

### 验收标准
- 完成一次考试并提交后，错题可在 `/api/wrong-book/review-queue` 查到。
- 错题本“今日复练清单”在 1 分钟内可见新增考试错题。
- 同题重复错不产生脏重复记录（按用户+题目去重更新）。

### 风险与回滚
- 风险：一次考试入队数量大导致写入波峰。
- 回滚：关闭考试提交后的自动入队，仅保留手动错题复练。

## 任务卡 3：教师预警一键动作

### 目标
- 教师在预警看板上可直接执行“修复动作”，减少从发现风险到干预落地的路径长度。

### 前端
- 教师预警列表增加一键按钮：
  - 一键布置复练
  - 一键发送提醒
  - 一键标记已处理
- 操作后实时刷新告警状态。

### 后端
- 新增动作接口（建议）：
  - `POST /api/teacher/alerts/:id/action`
  - `actionType`: `assign_review | notify_student | mark_done`
- 保留已有 `ack` 行为用于已读/确认。

### 数据
- 预警记录新增动作轨迹：
  - `lastActionType`
  - `lastActionAt`
  - `lastActionBy`
- 行为写入管理员日志用于审计。

### 验收标准
- 告警卡片点击动作后 3 秒内状态更新。
- 已执行动作可追溯（接口返回 + 管理日志可查）。
- 教师端流程从“发现风险”到“下发动作”不超过 2 次点击。

### 风险与回滚
- 风险：误操作导致过量布置任务。
- 回滚：动作入口增加二次确认，或仅保留 `mark_done`。
